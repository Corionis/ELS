Greetings from ${publisherName} at ${publisherRepo} :

Steps to complete the process of connecting to ${publisherRepo} with ELS:

 1. If ELS is not installed download from https://www.elsnavigator.com/, install and run it once so it creates it's directories.

    a. For Windows use the desktop shortcut to start ELS.
    b. For Linux and Mac run ELS-Navigator in your [Home]/.els/ directory.

 2. If ELS is running please close it.

 3. Unpack the attached file into your [Home]/.els/ directory. You may need to "show hidden files" to see it.

    a. Be sure the directory you select contains at least these subdirectories:
        • bin/
        • libraries/
        • local/
        • rt/
        • system/

 4. Start ELS. Goto System, Preferences and make sure "User-level security" is enabled on the General tab.

 5. Go to File, Open Publisher and open your "${userRepo}" library.

        a. If using Hints be sure to select Collection on the right, otherwise select Workstation.

 6. Adjust your ELS setup using the Libraries tab to adjust your General options, Locations and Bibliography as needed, see below.

        a. Do not change any Keys on the Libraries, General tab. They are used for login and to control ELS operations.

 7. Go to File, Open Subscriber and connect to "${publisherRepo}_connection". Be sure to select Remote Connection and Host on the right.

8. If using Hints:

    a. Go to File, Open Hint Keys to open hints.keys in the system/ subdirectory.
    b. Go to File, Open Hint Tracking to open the Hint Server in the libraries/ subdirectory. Be sure to select Remote Connection and Host on the right.


Other Notes

 A. The Libraries tab shows the definition of ${userRepo}, ${publisherRepo} and Hint Server if provided. Those tabs are:

    • General defines the basic options for ELS instances.
    • Locations specify minimum disk space for library sources.
    • Bibliography specifies libraries and their source(s).
    • User specifies Resources (ELS instances) and related permissions.

 B. Locations and Bibliography details for ${publisherRepo} are not provided for security reasons. Only connection options are used.

 C. Hint Keys and Hint Tracking are used to coordinate content renames, moves and deletes between multiple ELS instances (back-ups).

 D. For incoming connections you will need to adjust the NAT and/or port forwarding of your router.

 E. Subscriber LIstener(s):

    • The nightly back-up process uses one set of 3 sequential ports, e.g. 50271, 50272, 50273, with a dynamic Subscriber Listener.
        • A Custom IP with the first port is recommended, e.g. 140.82.112.4:50271
        • Use a cron job or Task Scheduler to schedule starting the listener.

    • For daily incoming connections a separate set of 3 sequential ports is needed with a persistent Subscriber Listener.
        • The Host ip:port of the Library, General tab is recommended so users don't have to enter a custom ip:port.
        • See Using systemd on Linux for ELS listeners at https://github.com/Corionis/ELS/discussions/176.
        • Or Add an app to run automatically at startup in Windows at https://support.microsoft.com/en-us/windows/configure-startup-applications-in-windows-115a420a-0bff-4a6f-90e0-1934c844e473.
