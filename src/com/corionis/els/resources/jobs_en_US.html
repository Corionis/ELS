Jobs

Jobs combine Tools into sequences of Tasks to define the connections required for each step. This allows a Tool to be reused with different Jobs.

 

Jobs execute defined Tasks in the sequence specified. A Task is a defined Tool with starting Origin(s) as arguments to the Tool.

Tasks

The Add button for Tasks displays a list of Tools that have been previously created. The list of Tools is updated each time the button is clicked.

Task Origins

Select the Library and connection type accordingly by clicking the "..." button. The options provided depend on the type of Task being defined.

  • "From: [name]" is only shown if a previous task's Origins are available.
  • "Any publisher" will use any loaded publisher
  • "Specific publisher" allows selection of a publisher from the list of libraries
  • "Any subscriber" will use any loaded subscriber
  • "Local subscriber" allows selection of a local subscriber from the list of libraries
  • "Remote subscriber (Host, Internet)" allows selection of a remote subscriber connecting on the Host address
  • "Remote subscriber (Listen, LAN)" allows selection of a remote subscriber connecting on the Listen address
  • "Remote subscriber (Custom)" allows entry of a remote subscriber connecting with a Custom server or IP address:port
  • "Local Hint Tracker" enables Hint processing using the current ELS instance.
  • "Remote Hint Server (Host, Internet)" enables Hint processing connecting on the Host address.
  • "Remote Hint Server (Listen, LAN)" enables Hint processing connecting on the Listen Address.

Remote Subscriber Listeners only listen on the Listen or Custom address. If the Listener value is not defined for that Library the Host value is used. Any Custom address must be valid.

 

The type of connection must be defined. Hint Servers only listen on the Listen address. If that value is not defined for that Library the Host value is used. Because there can be only one Hint Tracker/Server there is no Custom option as provided for remote Subscribers.

 

For Any publisher and Any subscriber Task Origins the current Publisher and/or Subscriber are used when the Job is executed.

  • If executed in the current instance using the Run ... button the currently-loaded publisher and/or subscriber are used by the Job.
  • If executed in a separate ELS instance from the command line, a shortcut, or using the Generate/Run ... button the publisher and subscriber arguments must be added if necessary.

The Origins Add button uses whatever is currently selected in the Browser. That means combinations can be assembled that do not necessarily make sense or will execute correctly. For instance selecting subscriber libraries in the Browser, but choosing a different subscriber for the Job that does not have matching libraries.

 

Some tasks, such as the Renamer, can use the Origins of a previous task. When previous Origins are available a "From: [name]" selection is added as the first entry in the ... combobox. That entry is the most recent task capable of providing Origins prior to the current task. A series of these tasks may be defined to perform complex operations.

Run ...

The Run button will execute the currently-selected Job in the currently-running ELS instance.

Generate/Run ...

Creates a command line to execute the currently-selected job.

 

Edit as desired. Select the Copy button to copy the command line to the clipboard. Select the Shortcut ... button to create a desktop shortcut, or the Run ... button to execute the Job in a new instance of ELS either in the foreground or background.

 

Note:

Depending on your choices additional parameters may be required for the Job to run correctly. In particular, if Any publisher or Any subscriber has been selected for any Tasks in the job then -p and/or -s parameters will be needed.
