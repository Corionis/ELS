Libraries

Libraries define parameters and options for an ELS library JSON file. There is one for each Publisher, Subscriber and a single Hint Tracker/Server if Hints are used (optional). Publisher and Subscriber are roles and may be reversed. The Navigator and command line are a Publisher that uses either a local or remote Subscriber.

 

Using Tools, Operations to create Tools that are put into Job tasks Subscriber and Hint Server Listeners may be defined so remote Publishers may connect.


General Tab

The General tab defines the key, communications parameters and other basic options. Different combinations of options are used depending on the role of the repository: Publisher, Subscriber or Hint Server.

  • Key
    A version 4 UUID. Every library must have a unique key. This key is also used for Authorization Keys and Hint Keys. Use the "..." button to generate a new key.
  • Host
    Host is the hostname:port or IPv4 address:port used by outside ELS systems to connect to this ELS instance.
  • Listen
    Listen is the hostname:port or IP address:port listener operations use to listen for incoming connections. If not defined Host is used. This setting is used with NAT or port forwarding to route incoming traffic to an inside system on a LAN. If it is the same IP as the server the port must be > +2 of the server port. Those two ports are used by the sftp component.
  • Time-out
    The number of minutes before a listen operation times-out and throws an error. A "heartbeat" is used during long back-up operations to avoid false time-outs.
  • Flavor
    The type of operating system where this library is located. Valid values: linux, mac or windows (only).
  • Case-sensitive
    Whether to handle directory and filenames as case-sensitive.
  • Temporary files dated
    Whether to add the current date and time to the end of temporary filenames. With dates files will accumulate. Without dates the same file will be overwritten on each use.
  • Temporary files location
    The directory path where temporary files are to be written.
  • Terminal Allowed
    Terminal allowed provides interactive access to an ELS instance when running in a listener mode. It is primarily used for diagnostic purposes to test the STTY command channel. The -a|--authorize [password] option is required to enable interactive access for the listener.
  • Ignore patterns
    A list of filenames to be ignored during back-up operations. Java regular expressions are supported. See Tools, Renamer help for details on regular expressions.

    Examples: (?i)desktop\.ini, Thumbs\.db, \.DS_Store
  • Email address
    Address to send fault emails when an error occurs and optional Mismatches and White's New reports.
  • Format
    Desired email format, HTML or text.
  • Skip if offline
    Skip sending emails and logging errors if the Subscriber failed to connect.
  • Mismatches
    Receive technical details of back-up operations.
  • What's New
    Receive user-oriented details of back-up operations.

Locations Tab

Locations tab defines devices or partial paths used to check for available free storage space.

 

Fully-qualified partial paths to their length should match the beginning of the full paths used in Library sources. Relative partial paths to their length should be contained within the full paths used in Library sources.

 

Examples:

Locations: Bibliography Sources:
D:\Media
E:\Media		 D:\Media\Movies
E:\Media\Movies
E:\Media\TV Shows
resources/media /mnt/drive01/resources/media
/mnt/drive02/resources/media

If a target for a backup, copy or move operation does not have a matching Location the default minimum free space is 10 GB.

 

A Target -t|-T [file] options was removed in Version 5. However the -t option is used to trigger a back-up operation from a Publisher.

 

A Hint Server only uses one Location, one Library with one Source for it's datastore.


Bibliography Tab

Bibliography tab defines libraries with one or more Source storage devices or paths.

 

The Match checkbox enables matching the dates of each Publisher and Subscriber file in the Library that have the same directory and filename. Example: Backing-up ELS configuration files.

 

The order of the sources defines the order searched for available free space, depending on the runtime setting of the "Back-fill" option.

 

User Tab

The User tab defines the type of user who owns this Library for either a Collection or Workstation. This tab is optional and may be enabled on the System, Preferences, General tab. This is an added level of security and provides a way to simplify the options shown in Navigator for non-administrators.

 

When the User-level security option is enabled Publishers login to Subscribers. The Subscriber's definition for the user overrides the user's definition of themselves. For remote Subscribers the user definition is retrieved from the Subscriber's copy of the Publisher's Library when the connection is made.

 

A Publisher login to a Subscriber can fail. A dialog is shown in Navigator and an error is written to the log file.

 

If a Subscriber does not have User-level security enabled the Publisher is considered to be an Administrator. However, if a Subscriber has User-level security enabled but the Publisher does not then login to the Subscriber will fail.

 

The User Type controls which options and functions are available.

  • Basic is intended for casual users showing simplified options.
  • Advanced is intended for personal-content Publishers.
  • Administrator has full control.

Resources define Grants when users access this or other Libraries.

 

Grants define read and write access to each individual library for the selected Library resource.

 

The Update button will update the list of available Libraries of the selected Resource when changes have been made.
